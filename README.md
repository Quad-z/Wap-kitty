# 🐱 [wap-kitty](https://github.com/Quad-z/Wap-kitty) <p align="center"> <img src="https://raw.githubusercontent.com/Quad-z/Wap-kitty/main/photos/Untitled(1).png" alt="wap-kitty Preview" width="100%"> </p> <p align="center"> <img src="https://img.shields.io/badge/Version-1.0.0-blue.svg?style=flat-square" alt="Version"> <img src="https://img.shields.io/badge/Python-3.10+-3776AB.svg?style=flat-square&logo=python&logoColor=white" alt="Python"> <img src="https://img.shields.io/badge/Terminal-Kitty-green.svg?style=flat-square" alt="Kitty"> <img src="https://img.shields.io/badge/License-MIT-yellow.svg?style=flat-square" alt="License"> </p> --- `wap-kitty` is a blazing-fast CLI automation utility that dynamically generates and applies beautiful, accessibility-first color schemes for the Kitty terminal based on your current desktop wallpaper. The project utilizes Google's advanced **Material 3 (M3) TonalSpot** algorithm to calculate precise tonal palettes. Unlike traditional generators (such as `pywal`) that blindly grab raw pixels or average out colors, `wap-kitty` relies on the mathematical rigor of the **HCT (Hue/Chroma/Tone)** color space. Upon execution, it instantly flashes the terminal configuration live and prints out the active color palette in `stdout` using clean, True Color visual blocks. --- ## ✨ Features * 🧠 **M3 TonalSpot Engine:** Native generation of Primary, Secondary, Tertiary, Neutral, and Error palettes using Google's official C++ bindings via `pybind11`. * 👁️ **Readability First:** Strict contrast-ratio enforcement (target 3.0) ensures your terminal text never blends illegibly into the background. * 🎨 **Deep UI Mapping:** Goes way beyond basic 16 ANSI overrides—it dynamically themes your background, foreground, cursor, active selections, window borders, and tab bars. * ⚡ **Performance Optimized:** Images are automatically downscaled to $128 \times 128$ before running a hardware-accelerated k-means clustering process for instant execution. * 🔄 **Zero-Click Automation:** Native systemd path units track wallpaper swaps in `noctalia` on the fly, instantly matching your terminal theme without lifting a finger. --- ## 📂 Project Structure ```text wap-kitty/ ├── wap # Bash script entry point (manages PYTHONPATH) ├── install.sh # Automated dependency installer & symlinker ├── README.md # Project documentation ├── .gitignore # Ignores __pycache__/ and *.pyc ├── systemd/ │ ├── wap-kitty.path # Systemd Path unit (monitors config changes) │ └── wap-kitty.service# Systemd Service unit (triggers the generator) └── wap_kitty/ ├── __init__.py # Version metadata (1.0.0), exports main ├── __main__.py # Script execution entry (python -m wap_kitty) └── core.py # Core logic (Wallpaper detection, M3, config engine — 157 lines) 
``` --- ## 🛠 Installation & Setup ### 1. Clone the Repository ```bash git clone [https://github.com/Quad-z/Wap-kitty.git](https://github.com/Quad-z/Wap-kitty.git) cd Wap-kitty ``` ### 2. Run the Installer You can use `install.sh` or quickly handle the setup manually: ```bash pip install --user --break-system-packages material-color-utilities Pillow ln -sf "$(pwd)/wap" ~/.local/bin/wap ``` ### 3. Configure Kitty Add this line to your `~/.config/kitty/kitty.conf` to enable live theme loading: ```kitty include current-theme.conf ``` > ⚠️ **Note:** Your personal configuration values like `background_opacity` and `background_blur` are safely preserved and will not be overwritten by `wap-kitty`. ### 4. Enable Automated Hooks (Optional) If you manage your desktop wallpapers using `noctalia`, seamlessly activate real-time background tracking: ```bash cp systemd/wap-kitty.{service,path} ~/.config/systemd/user/ systemctl --user daemon-reload systemctl --user enable --now wap-kitty.path ``` --- ## 💡 Usage ```bash # Auto-detect current desktop wallpaper from ~/.cache/noctalia/wallpapers.json wap # Force-generate a theme template from a specific image file wap ~/Pictures/my_wallpaper.jpg # Quick test run using paths wap ~/Downloads/wap-kitty/photos/Untitled(1).png Imba ``` --- ## ⚙️ Technical Specifications & Core Architecture ### Material 3 → Kitty ANSI Mapping Every generated `TonalPalette` provides a `get_argb(tone)` method where `tone` ranges from 0 to 100. The script strips alpha channels (`& 0xFFFFFF`) and strictly routes M3 architectural roles directly into Kitty variables: | Kitty Token | M3 Palette Source | Kitty Token | M3 Palette Source | | :--- | :--- | :--- | :--- | | **background** | `primary` | **color0 / color8** | `neutral` | | **foreground** | `neutral` | **color1 / color9** | `error` | | **cursor / text** | `primary` / `primary` | **color2 / color10** | `secondary` | | **selection (bg/fg)**| `primary` / `neutral` | **color3 / color11** | `tertiary` | | **url_color** | `primary` | **color4 / color12** | `primary` | | **active_border** | `primary` | **color5 / color13** | `tertiary` | | **inactive_border**| `neutral` | **color6 / color14** | `secondary` | | **active_tab (fg/bg)**| `background` / `primary`| **color7 / color15** | `neutral` | | **inactive_tab (fg/bg)**| `neutral` / `neutral` | | | ### TonalSpot Quantization Logic Under the hood, `material-color-utilities` calculates a strict `Variant.TONALSPOT` layout profile: * **Chroma Constraints:** Primary = 36, Secondary = 16, Tertiary = 24, Neutral = 6, Neutral Variant = 8. * **Hue Shifts:** Secondary tracks the seed hue, Tertiary = Hue + 60°, Error = 25°. * Export targets are hit simultaneously: writing straight to `~/.config/kitty/current-theme.conf` for the active layout and duplicating to `~/.cache/wal/colors-kitty.conf` to guarantee deep backward compatibility with old-school `pywal` pipelines. --- ## 📄 License This project is open-source and released under the **MIT License**.
